01 IntelliJ Swing UI Form 이란?

IntelliJ IDEA에는 Swing 컴포넌트를 드래그 앤 드롭으로 배치할 수 있는 GUI Form Designer가 내장되어 있습니다. 디자이너에서 UI를 구성하면 .form(XML) 파일이 자동 생성되고, 연결된 .java 클래스에 필드 바인딩이 선언됩니다.

코드로 직접 new JTextField()를 작성하는 대신, 시각적 편집기에서 컴포넌트를 배치하고 속성을 설정하면 런타임에 자동으로 UI가 구성됩니다.

📝 코드 직접 작성 방식
JPanel panel = new JPanel();
panel.setLayout(
  new BorderLayout());
JTextArea ta = new JTextArea();
JTextField tf = new JTextField();
JButton btn = new JButton("전송");
panel.add(ta, BorderLayout.CENTER);
...
→
🎨 UI Form Designer 방식
// .form 파일이 레이아웃 담당
// 필드 선언만 클래스에 작성
private JPanel panel1;
private JTextArea textArea1;
private JTextField textField1;
private JButton 전송Button;
// → 드래그로 배치, 속성은 GUI에서
💡 .form 파일과 .java 파일은 항상 쌍으로 존재합니다. Main.form이 레이아웃 정보를 담고, Main.java가 이벤트 로직을 담습니다.
02 UI Form 생성 방법 (IntelliJ)
① 새 GUI Form 파일 생성
패키지 우클릭 → New › Swing UI Designer › GUI Form 선택

Form name에 Main 입력,
Base layout manager: BorderLayout 선택
② 컴포넌트 배치 (Design 탭)
좌측 Palette에서 컴포넌트를 드래그

• JTextArea → Center 영역에 배치
• JTextField + JButton → South 패널에 배치

각 컴포넌트의 field name은 우측 Properties에서 설정
③ Java 클래스 연결 확인
Main.java에 자동 생성된 필드 확인:

private JPanel panel1;
private JTextArea textArea1;
private JTextField textField1;
private JButton 전송Button;

생성자에서 add(panel1)으로 패널 추가
⚠️ 주의: UI Form으로 생성된 필드는 직접 new로 초기화하지 않습니다. .form XML이 런타임에 컴포넌트를 생성해 필드에 주입합니다. 수동으로 textArea1 = new JTextArea()를 작성하면 충돌이 발생할 수 있습니다.
03 Main.form XML 구조 분석

.form 파일은 IntelliJ가 자동 관리하는 레이아웃 설계도입니다. 직접 수정하지 않고 Design 탭에서 편집하는 것을 권장합니다.

🗂 Main.form — 컴포넌트 계층 구조
<form bind-to-class="...Main">
<grid binding="panel1" layout-manager="BorderLayout"> ← 루트 패널 (BorderLayout)
<grid layout-manager="FlowLayout"> ← South 패널
<constraints border-constraint="South"/>
<component class="JTextField" binding="textField1" columns="25"/>
<component class="JButton" binding="전송Button" text="전송"/>
</grid>
<scrollpane> ← Center 영역 (JScrollPane)
<constraints border-constraint="Center"/>
<component class="JTextArea" binding="textArea1" lineWrap="true"/>
</scrollpane>
</grid>
</form>
XML 속성역할설명
bind-to-class 연결 이 .form이 어떤 Java 클래스와 연결되는지 지정. Main.java의 필드와 바인딩됨
binding="panel1" 연결 Java 클래스의 private JPanel panel1 필드에 이 컴포넌트를 주입
layout-manager="BorderLayout" 레이아웃 루트 패널의 레이아웃을 BorderLayout으로 설정. Center/South 영역 분할
border-constraint="South" 레이아웃 이 컴포넌트를 BorderLayout의 SOUTH 위치에 배치하도록 지정
border-constraint="Center" 레이아웃 JScrollPane(JTextArea)를 가운데(남은 공간 전체)에 배치
layout-manager="FlowLayout" 레이아웃 South 패널 내부는 FlowLayout — JTextField + JButton을 가로로 나열
columns="25" 속성 JTextField의 표시 너비를 25자 기준으로 설정
lineWrap="true" 속성 JTextArea에서 긴 줄이 자동으로 줄 바꿈되도록 설정

05 소스 코드 — Main.java
Main.java UI Form 연동 
package inhatc.cse.java.chap17.ex0_frame;

import javax.swing.*;
import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;

public class Main extends JFrame {

    // ── UI Form과 바인딩되는 컴포넌트 필드 ──────────────────────
    // .form 파일이 런타임에 이 필드들에 컴포넌트를 자동 주입
    // ⚠ new로 직접 초기화하지 않는다!
        private JTextArea  textArea1;
        private JPanel     panel1;
        private JTextField textField1;
        private JButton    전송Button;

    public Main() {
        // ── JFrame 기본 설정 ──────────────────────────────────────
        setTitle("프레임 연습");
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);  // X 버튼 클릭 시 종료
        setSize(400, 300);                              // 창 크기 설정
        setLocationRelativeTo(null);                     // 화면 중앙 배치
        setVisible(true);                                // 창 표시

        // ── UI Form 패널을 JFrame에 추가 ─────────────────────────
                add(panel1);   // .form에서 생성된 루트 패널을 JFrame에 등록

        // ── 방법 1: 익명 내부 클래스로 이벤트 처리 (기본 방식) ───
        전송Button.addActionListener(new ActionListener() {
            @Override
            public void actionPerformed(ActionEvent e) {
                String text = textField1.getText();     // 입력창 텍스트 가져오기
                textArea1.append(text + "\n");        // 텍스트창에 추가
                textField1.setText("");               // 입력창 초기화
            }
        });

        // ── 방법 2: 람다식으로 동일 이벤트 처리 (Java 8+) ────────
//        전송Button.addActionListener(e -> {
//            String text = textField1.getText();
//            textArea1.append(text + "\n");
//            textField1.setText("");
//        });
    }

    public static void main(String[] args) {
        new Main();    // 생성자 호출 → JFrame 창 표시
    }
}
코드분류설명
extends JFrame 구조 JFrame을 상속해 창 기능을 직접 보유. setTitle(), setSize() 등을 this. 없이 호출 가능
private JTextArea textArea1 등 Form .form의 binding 속성과 이름이 일치해야 함. UI Designer가 런타임에 자동 주입 — new 불필요
setLocationRelativeTo(null) 구조 null을 전달하면 화면 정중앙에 창을 배치. 이전 예제의 setLocation(x, y)와 달리 자동 계산
add(panel1) Form .form이 주입한 루트 패널을 JFrame의 콘텐츠 영역에 추가. 이 한 줄로 전체 UI가 표시됨
textField1.getText() 이벤트 JTextField의 현재 입력 내용을 String으로 반환. 버튼 클릭 시 이 값을 읽어 처리
textArea1.append(text + "\n") 이벤트 JTextArea 끝에 텍스트를 추가. "\n"으로 줄 바꿈. setText()는 전체 교체, append()는 누적 추가
textField1.setText("") 이벤트 전송 후 입력창을 비워 다음 입력 준비. UX 필수 처리
06 ActionListener 이벤트 처리 — 2가지 방식 비교

버튼 클릭 이벤트를 처리하는 방법은 크게 두 가지입니다. 익명 내부 클래스 방식과 람다식 방식 모두 동일하게 동작하며, 현대 Java(8+)에서는 람다식이 더 간결합니다.

📜
익명 내부 클래스 방식
new ActionListener() { @Override actionPerformed(){...} }

Java 7 이하에서 사용. 코드가 길지만 명시적으로 무슨 일이 일어나는지 잘 보임. 학습 초기에 인터페이스 구조를 이해하는 데 적합
⚡
람다식 방식 (Java 8+)
e -> { ... }

함수형 인터페이스(메서드 1개)이므로 람다로 표현 가능. 코드가 짧고 가독성이 높음. 실무에서 권장
📜 익명 내부 클래스
전송Button.addActionListener(
  new ActionListener() {
    @Override
    public void actionPerformed(
        ActionEvent e) {
      String text =
        textField1.getText();
      textArea1.append(
        text + "\n");
      textField1.setText("");
    }
  }
);
⚡ 람다식 (Java 8+)
전송Button.addActionListener(
  e -> {
    String text =
      textField1.getText();
    textArea1.append(
      text + "\n");
    textField1.setText("");
  }
);
✅ 두 방식은 완전히 동일하게 동작합니다. ActionListener는 actionPerformed() 메서드 하나만 가진 함수형 인터페이스이기 때문에 람다식으로 대체할 수 있습니다.
07 실행 흐름 — 버튼 클릭 시 동작 순서
1
main() 실행 → new Main()
public static void main()에서 new Main() 호출. 생성자에서 setTitle(), setSize(), setVisible(true)를 실행해 창이 화면에 표시됨. .form이 자동으로 컴포넌트를 생성하고 필드에 주입
2
add(panel1) — 루트 패널 등록
.form이 주입한 panel1(BorderLayout)을 JFrame에 추가. CENTER에 JScrollPane(JTextArea), SOUTH에 JTextField + JButton이 배치된 전체 UI가 한 번에 표시됨
3
ActionListener 등록 — 전송 버튼
전송Button.addActionListener(...)로 이벤트 리스너 등록. 이 시점에는 실행되지 않음 — 버튼이 클릭될 때까지 대기 상태
4
사용자가 텍스트 입력 후 [전송] 클릭
JVM이 actionPerformed(ActionEvent e)를 자동 호출. textField1.getText()로 입력 내용 읽기 → textArea1.append(text + "\n")로 텍스트창에 추가 → textField1.setText("")로 입력창 초기화
5
반복 대기 — 이벤트 루프
Swing은 내부적으로 EDT(Event Dispatch Thread)가 이벤트 루프를 유지. 창이 닫히지 않는 한 버튼 클릭을 계속 감지하고 actionPerformed()를 반복 호출
08 핵심 포인트 정리
🗂
.form 파일 역할
레이아웃 설계도. 직접 편집하지 않고 Design 탭에서 편집. .java와 항상 쌍으로 존재
🔗
바인딩(Binding)
.form의 binding="textArea1"과 Java 필드명이 정확히 일치해야 주입이 동작함
🚫
new 초기화 금지
.form이 주입하는 필드를 new JTextArea()로 직접 초기화하면 충돌 발생. 선언만 하고 초기화는 Form에 맡길 것
📤
append vs setText
append()는 기존 내용 유지하며 추가, setText()는 전체 교체. 채팅창은 append(), 입력창 초기화는 setText("")
⚡
람다 vs 익명 클래스
ActionListener는 함수형 인터페이스이므로 람다 사용 가능. 두 방식 동작 동일 — 람다가 더 간결
🏗
add(panel1) 순서
setVisible(true) 전에 add(panel1) 호출 권장. 또는 호출 후 revalidate() / repaint() 필요할 수 있음
⚠️ 이 예제의 한계: setVisible(true)를 add(panel1)보다 먼저 호출하고 있습니다. 일반적으로 모든 컴포넌트 추가 후 setVisible(true)를 호출하는 것이 권장됩니다. 동작에는 문제없지만, 코드 순서를 바꿔도 결과가 같은지 직접 실험해보세요.
