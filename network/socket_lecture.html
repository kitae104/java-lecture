소켓 통신 Server / Client

java.net 패키지를 이용한 TCP 소켓 기반 서버-클라이언트 양방향 채팅 프로그램

java.net.Socket java.net.ServerSocket BufferedReader BufferedWriter TCP / Port 9999
01 소켓 통신 개요

소켓(Socket)은 네트워크에서 프로세스 간 통신을 위한 양쪽 끝단(End-point)입니다. Java는 java.net 패키지에 Socket과 ServerSocket 클래스를 제공하여 TCP 기반의 신뢰성 있는 연결 통신을 구현할 수 있습니다.

서버는 특정 포트에서 클라이언트의 접속을 대기(accept())하고, 클라이언트는 서버 IP와 포트 번호로 연결을 요청(new Socket(ip, port))합니다. 연결이 성립되면 양쪽 모두 InputStream / OutputStream으로 데이터를 주고받습니다.

03 소스 코드
Server.java 
package inhatc.cse.java.chap17.ex2_serverclient;

import java.io.*;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.Scanner;

public class Server {
    public static void main(String[] args) {
        ServerSocket server = null;
        Socket socket = null;
        BufferedReader in = null;
        BufferedWriter out = null;
        Scanner sc = new Scanner(System.in);

        try {
            // 서버 소켓 생성 - 포트 9999에서 대기
            server = new ServerSocket(9999);
            System.out.println("연결 대기중.....");

            // 클라이언트 연결 수락 (블로킹)
            socket = server.accept();
            System.out.println("연결 되었습니다.");

            // 스트림 획득 (문자 기반 버퍼 스트림으로 래핑)
            in  = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            out = new BufferedWriter(new OutputStreamWriter(socket.getOutputStream()));

            while(true) {
                // 클라이언트 메시지 수신
                String inMsg = in.readLine();

                // "bye" 수신 시 종료
                if(inMsg.equalsIgnoreCase("bye")) {
                    System.out.println("클라이언트가 나갔습니다.");
                    break;
                }

                // 정상 메시지 출력 후 응답 전송
                System.out.println("클라이언트 : " + inMsg);
                System.out.print("보내기 >>");
                String outMsg = sc.nextLine();
                out.write(outMsg + "\n");
                out.flush();  // 버퍼를 강제로 비워 전송
            }

        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            try {
                // 리소스 해제 (역순으로)
                sc.close();  out.close();  in.close();
                socket.close();  server.close();
            } catch (IOException e) { e.printStackTrace(); }
        }
    }
}
코드 구분 설명
new ServerSocket(9999) SERVER 포트 9999번에 서버 소켓을 바인딩. 운영체제에 해당 포트를 점유 요청
server.accept() SERVER 클라이언트가 연결할 때까지 블로킹(Blocking). 연결되면 통신용 Socket 객체 반환
socket.getInputStream() STREAM 수신용 스트림. BufferedReader로 래핑해 readLine() 사용 가능
socket.getOutputStream() STREAM 송신용 스트림. BufferedWriter로 래핑해 문자열 쓰기 편의성 확보
out.flush() STREAM 버퍼에 쌓인 데이터를 즉시 전송. 반드시 호출해야 상대방이 수신 가능
inMsg.equalsIgnoreCase("bye") SERVER 종료 신호 감지. 대소문자 무관 비교. 감지 시 while 루프 탈출 후 리소스 해제
Client.java 
package inhatc.cse.java.chap17.ex2_serverclient;

import java.io.*;
import java.net.InetSocketAddress;
import java.net.Socket;
import java.util.Scanner;

public class Client {
    public static void main(String[] args) {

        Socket socket = null;
        BufferedReader in = null;
        BufferedWriter out = null;
        Scanner sc = new Scanner(System.in);

        try {
            // 서버에 연결 (IP: localhost, Port: 9999)
            socket = new Socket("localhost", 9999);

            // 스트림 획득 (문자 기반 버퍼 스트림으로 래핑)
            in  = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            out = new BufferedWriter(new OutputStreamWriter(socket.getOutputStream()));

            while(true) {
                System.out.print("보내기 >>");
                String outMsg = sc.nextLine();

                // "bye" 입력 시 서버에 알리고 종료
                if(outMsg.equalsIgnoreCase("bye")) {
                    out.write(outMsg + "\n");
                    out.flush();
                    break;
                }

                // 정상 메시지 전송
                out.write(outMsg + "\n");
                out.flush();

                // 서버 응답 수신 및 출력
                String inMsg = in.readLine();
                System.out.println("서버 >> : " + inMsg);
            }

        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            try {
                // 리소스 해제
                sc.close();  out.close();
                in.close();  socket.close();
            } catch (IOException e) { e.printStackTrace(); }
        }
    }
}
코드 구분 설명
new Socket("localhost", 9999) CLIENT 서버 IP와 포트로 연결 요청. 서버가 accept() 중이어야 성공. 연결 실패 시 IOException
socket.getOutputStream() STREAM 서버로 데이터를 보내는 스트림. 클라이언트의 Output = 서버의 Input
socket.getInputStream() STREAM 서버에서 데이터를 받는 스트림. 클라이언트의 Input = 서버의 Output
out.write(outMsg + "\n") CLIENT "\n"을 반드시 포함. 상대방 readLine()이 줄바꿈 문자를 경계로 한 줄을 읽기 때문
in.readLine() STREAM 서버가 응답할 때까지 블로킹. 상대방이 "\n"을 보내야 반환됨
equalsIgnoreCase("bye") CLIENT 종료 명령 처리. 서버에 "bye"를 전송 후 루프 탈출 → 스트림/소켓 자원 해제
04 프로그램 실행 순서
1
서버 먼저 실행
Server.java를 먼저 실행. "연결 대기중....." 메시지가 출력되며 accept()에서 블로킹 상태 진입
2
클라이언트 실행
Client.java 실행 → new Socket("localhost", 9999)로 서버에 접속. 서버 측에 "연결 되었습니다." 출력
3
클라이언트에서 메시지 입력
클라이언트 콘솔에 보내기 >> 프롬프트가 표시됨. 메시지 입력 후 Enter → 서버로 전송
4
서버에서 응답 입력
서버 콘솔에 수신 메시지 출력 후 보내기 >> 프롬프트 표시. 응답 입력 → 클라이언트로 전송
5
종료 — bye 입력
클라이언트에서 bye 입력 시 서버에 종료 신호 전달 → 양쪽 모두 finally 블록에서 리소스 해제
05 핵심 포인트 & 주의사항
🔄
스트림 방향
클라이언트의 OutputStream ↔ 서버의 InputStream. 반드시 서로 반대 방향으로 연결됨
⚠️
flush() 필수
BufferedWriter는 버퍼가 가득 차야 전송. 즉시 전송하려면 flush()를 반드시 호출
↩️
개행 문자 "\n"
readLine()은 "\n"을 기준으로 한 줄을 읽음. write 시 반드시 "\n" 포함
🔒
블로킹 I/O
현재 코드는 단일 스레드. 상대가 응답하지 않으면 readLine()에서 무한 대기
♻️
리소스 해제
finally 블록에서 스트림 → 소켓 순서로 해제. server.close()도 잊지 않도록
🚀
실행 순서
반드시 서버 먼저 실행. 클라이언트가 먼저 실행되면 Connection refused 예외 발생
